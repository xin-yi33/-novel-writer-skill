# Novel Writer Skill — 小说写作助手

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

一个 Claude Code Agent Skill，帮助网文作者从细纲到完整章节进行创作。支持本地写作、长篇写作记忆管理、在线发布到番茄小说平台，**内置中文写作质量指南（章节结构、对话技巧、标题创作、节奏控制等）**。

---

## 功能概览

| 功能 | 说明 |
|------|------|
| **写新章节** | 读取细纲 + 小说设定 → 按写作质量标准生成完整章节 → 保存到本地或发布到番茄 |
| **长篇写作模式** | 配套记忆管理系统（bible / summary / recent / status / 人物卡），支持跨 session 续写 |
| **人物卡管理** | 基于人物卡的一致性校验，自动检测性格偏差并向作者确认 |
| **卡文助手** | 分析作者已有章节的文风，提供 2-3 个不同方向的简写示例（300-800字）供选择 |
| **发布到番茄** | 通过 MCP 浏览器自动上传到 fanqienovel.com，支持多章分次存草稿 |
| **续写** | 自动读取最新章节衔接生成下一章，卡文时自动转卡文助手 |
| **标题创作建议** | 基于 J3 节标题创作指南（题材-风格映射 + 五种创作技巧），生成 5-8 个候选标题 |
| **标题优化** | 分析已有标题的优缺点，提供优化方向和修改建议 |
| **细纲管理** | 新建、查看、修改细纲文件 |
| **进度查看** | 展示章节列表、字数统计、写作进度 |
| **写作质量参考** | 内置完整写作指南，AI 生成时自动运用并自检 |

### 写作质量标准覆盖

| 质量维度 | 内容 |
|---------|------|
| **章节结构** | 标准四段结构（开头钩子 20% → 发展推进 60% → 高潮 20% → 结尾钩子）、十种开头技巧、章节类型分类与字数指导 |
| **对话写作** | 六种对话目的、简洁与真实原则、潜台词技巧、对话与动作结合、快/慢/沉默三种节奏、六种场景类型、常见问题规避 |
| **标题创作** | 六种题材风格映射、五种创作技巧（核心冲突提炼、主角命名、意象隐喻、反差、悬念留白）、质量标准与禁忌 |
| **中文文学技法** | 白描、留白、意象营造、草蛇灰线、蒙太奇剪辑——传统文学技法的可操作指南 |
| **节奏控制** | 句子节奏（长短句交替）、段落呼吸（段落长度控制）、信息密度波浪（高→低→高→低交替） |
| **打破预期** | 四种技术：预期反转、信息差利用、复杂动机、节外生枝 |
| **质量检查清单** | 八大维度逐项自查（开头/内容/人物/对话/节奏/语言文字/连贯性/题材特定） |
| **题材特定检查** | 悬疑/犯罪、言情/都市、奇幻/科幻、动作类专属检查项 |


### 手动使用

直接对 Claude 说以下关键词即可自动触发（无需额外安装）：

- 写小说 / 生成小说章节 / 帮我写小说
- 根据细纲生成文章

- 上传到番茄 / 查看进度 / 管理细纲
- 想个标题 / 帮我起个书名（参考标题创作指南）

## 长篇写作模式与记忆系统详解

适用于连载超过 5 章的长篇小说。通过**内存文件系统**解决 AI 上下文窗口限制，即使关闭对话再重新开始，也能从断点精确续写。

### 为什么要用记忆系统？

AI 的上下文窗口有限，无法一次性容纳整部小说。每次对话开始时，AI 只加载最少但最关键的信息：

| 文件 | 作用 | 大小限制 | 更新频率 |
|------|------|---------|---------|
| **bible.md** | 世界观核心设定（力量体系、不可违背的规则） | < 2000 字 | 仅作者确认时更新 |
| **summary.md** | 每章自动压缩的摘要 | 自动管理 | 每写完一章更新 |
| **recent.md** | 最近 2 章全文（保持风格和情绪连贯） | < 6000 字 | 每写完一章替换 |
| **status.md** | 当前进度 + 未解决/已回收伏笔 + 人物情绪 | 轻量 | 每写完一章更新 |
| **人物卡** | 出场角色的性格和行为约束 | 每人约 200 字 | 按需更新 |

### summary 自动压缩机制

为防止摘要文件无限膨胀，系统内置自动压缩算法：

```
单章压缩：compressed = max(原字数 / 10 - 100, 50)
示例：3000 字 → 200 字，1000 字 → 50 字
```

**总纲合并：** 每积累 30 条单章摘要，自动将每 5 章合并为一条总纲，保留核心事件和转折点。

**文件轮转：** 当总纲填满文件后，自动开启新的 summary 文件（summary2.md → summary3.md → ...），永不丢失历史。

### 典型工作流

```
会话1：写 ch01-ch05 → 内存文件自动更新
会话2（重新开始）：AI 读取 bible + summary + recent + status → 从 ch06 无缝续写
会话3：继续写 ch11 → 摘要满30条 → 自动压缩为总纲
...
```

> 这种模式意味着即使写 500 章，AI 也能精确知道前情提要，不会出现"写后面忘前面"的问题。

## 快速开始

1. 对 Claude 说"我要写小说"
2. 填写 `novel/requirements.md` 中的小说设定
3. 创建第一个细纲文件放入 `novel/outlines/`
4. 开始写作！

AI 会在生成章节时自动运用内置的写作质量标准（章节结构、对话技巧、节奏控制等），并在交付前进行质量自检。

## 卡文助手

当你说"卡文了"，技能会自动：

1. 读取你最近 3-5 章，分析你的文风特点
2. 询问具体的卡文点（情节走向 / 细节卡壳 / 情绪不对 / 过渡困难 / 开篇困难）
3. 生成 2-3 个不同方向的 300-800 字简写示例（开篇困难时参考 J1 十种开头技巧）
4. 选中方案后可扩展为完整章节
